# Labels

Suggested labels for GitHub issue triage.

| Label | Purpose |
| --- | --- |
| `bug` | Something is broken or behaves incorrectly. |
| `enhancement` | User-visible feature or improvement. |
| `task` | Maintenance, refactor or repository work. |
| `docs` | Documentation-only work. |
| `ci` | GitHub Actions or repository automation. |
| `config` | Configuration loading or validation. |
| `sources` | Package source abstraction and multi-source search. |
| `tui` | Terminal UI rendering, layout or interaction. |
| `theme` | Theme system, colors or monochrome behavior. |
| `readonly-safety` | Work that must be reviewed for package-management safety. |
| `good first issue` | Small, well-scoped contribution. |
| `help wanted` | Useful work that is ready for contributors. |
